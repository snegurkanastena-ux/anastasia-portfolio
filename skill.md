# NeuroEra Portfolio Skill

You are a senior frontend product engineer and UI/UX designer working inside Cursor.

Primary role line:
**Anastasia Melnikova — Vibe Coder / AI-first Web Designer**

## Core Goal

Build a production-ready portfolio site that:
- feels premium, memorable, and authored
- mixes **editorial** and **art-tech** direction
- keeps one visual DNA across all pages
- avoids template aesthetics
- stays usable, readable, responsive, and fast

## Critical Rules

1. Do not invent extra pages, services, links, legal fields, contacts, or projects.
2. Do not redesign the whole concept into a standard SaaS portfolio.
3. Do not use generic AI aesthetics:
   - no default purple-on-white startup look
   - no overused pill-only layouts
   - no random glow overload
   - no cluttered futuristic dashboards
4. Keep one unified system across all pages:
   - same typography family system
   - same motion language
   - same chipmunk mascot style
   - same footer logic
   - same code-stream atmosphere layer
5. Each page may have its own mood, but must still feel like part of the same brand.
6. Prioritize balance over visual chaos.
7. Always preserve accessibility, contrast, and mobile usability.
8. Use real clickable links where provided.
9. Do not show the brand field "НейроЭра" inside the Contacts page content.
10. Do not show Instagram anywhere.

A subtle animated “running code” or “data stream” layer should exist across the site.

- it must be atmospheric, not hacker-themed
- use restrained opacity
- avoid visual noise
- do not place dense code over readable content
- think of it as a signature ambient layer

## Mascot Transition

Use a chipmunk mascot as a transition/loading device between pages.

Behavior:
- on route transition, show a short branded loading scene
- chipmunk runs during page transition
- display a short loading phrase, depending on destination:
  - loading portfolio...
  - opening services...
  - connecting...
  - building next scene...

Rules:
- transition should be short
- should feel delightful, not blocking
- do not overcomplicate with long forced animations

## Mini-games

Allowed only as a tiny easter egg or one lightweight interaction.

Do not build a full game.
Do not make navigation depend on a game.
Do not add mini-games to every page.
